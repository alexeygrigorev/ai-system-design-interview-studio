# UX Jobs To Be Done

This product is for candidates practicing AI engineering system design interviews.

## Primary Jobs

1. When I start a practice session, I want to choose a realistic AI engineering prompt, level, interviewer style, and constraints so the session matches the interview I am preparing for.
2. When I explain an architecture, I want to draw components, data flows, notes, and boundaries quickly so I can reason visually without fighting the tool.
3. When the interviewer probes my answer, I want one clear question at a time so I can practice realistic interview pacing.
4. When I finish, I want evidence-based feedback tied to the transcript and diagram so I know what to improve.

## Supporting Jobs

- Save and resume a practice session.
- Export a transcript, diagram, and final assessment.
- Compare attempts across topics and seniority levels.
- Practice under different pressure levels without changing the rubric.
- See where my answer missed production AI concerns such as evaluation, privacy, monitoring, cost, latency, and fallback behavior.

## UX Principles

- Put the working surface first: interview, diagram, and answer input are the core screen.
- Keep controls dense and predictable because this is a repeated practice tool.
- Avoid teaching text inside the app unless it directly helps the current task.
- Make feedback actionable and evidence-based, not generic encouragement.
- Treat diagrams as supporting evidence for architecture reasoning, not as decoration.
